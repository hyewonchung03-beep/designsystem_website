Action Priority

화면에서 가장 중요한 핵심 액션에 사용합니다. 화면당 하나의 버튼 사용을 권장합니다.

Primary 액션을 보완하거나 대체할 때, 중요도가 유사한 대안 액션에 사용합니다.

추가 옵션 등 보조적인 기능에 사용하며, 화면 내 시각적 우선순위를 최소화합니다.

Placement

Primary Action은 중요한 행동 1개만 강조합니다.

Do

가장 중요한 작업에 더 눈에 띄는 버튼을 선택하세요. 예를 들어 텍스트 버튼 옆에 채워진 버튼을 배치할 수 있습니다.

Do

강조 표시가 강한 동작 옆에 강조도가 낮은 동작을 옆에 배치하세요.

페이지 레벨 CTA 배치

Do

페이지 전체에 영향을 끼치는 CTA는 핵심 액션 ‘Primary’ 버튼 컬러를 사용하며 페이지 헤더의 우측 영역에 배치됩니다.
그 외 보조 액션은 ‘Secondary’ 버튼 컬러를 사용하여 시각적으로 구분합니다.

Don’t

주요 액션이 2개 이상이면 사용자가 혼란을 느낄 수 있으므로,
‘Primary’ 버튼을 여러개 사용하지 않습니다.

UX Guidelines

버튼 아이콘 사용 가이드

Do

Chevron 아이콘은 우측에, 그 외 아이콘은 좌측에 사용합니다.

Don’t

아이콘 좌측 외 사용은 지양합니다.

레이블 텍스트

Do

필요한 만큼 텍스트를 줄이며 추가적인 맥락을 제공할 경우에
아이콘을 포함합니다.

Don’t

텍스트 줄 바꿈이나 잘림을 피하십시오.

Do

Table에서 버튼 사용시, 버튼 크기를 동일하게 사용합니다.

Don’t

라벨의 길이가 급격하게 길어지거나 짧아지지 않도록 합니다.

Do

버튼에 사용하는 문구와 제목 및 기타 관련 텍스트에 사용하는 문구가
동일해야 합니다.

Don’t

같은 의미의 액션에 다른 레이블을 사용하지 않습니다.

Right-to-left languages

텍스트를 줄이고, 필요 시 아이콘으로 보완합니다.

RTL언어에서 아이콘은 UI 레이아웃을 좌우반전하여 제공합니다.

아이콘(선택)

Do

아이콘은 선택적으로 텍스트 보조하는 용도로만 사용합니다.

Don’t

버튼 중앙에 아이콘과 텍스트를 세로로 정렬하지 않습니다.

Danger Action 버튼 사용 가이드

Do

롤백 불가, 로그 삭제, DB 초기화, 히스토리 초기화 등 해당 액션으로 시스템/데이터에 중대한 손실일 경우에 Danger 버튼을 사용합니다.

Don’t

Retry, Cancel, Stop 등 복구 가능한 행동이나 영향도가 낮은 액션인 경우 사용하지 않습니다. (‘Primary’ 버튼 컬러를 사용합니다.)
